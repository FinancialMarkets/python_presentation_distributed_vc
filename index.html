GitHub/GitLab

Sites such as GitHub and GitLab allow you to host git repositories online. This is useful to us for two main reasons:

  • We can collaborate with others.
  • We can show others our work.

Collaboration

By having one central repository online, many people can collaborate on the same set of documents.

  • Everyone makes to their local copies.
  • They push their changes to the online repository.
  • They pull others' changes to their local repository.

We'll discuss how this is done later.

An Example

Linux Kernel development is done using distributed version control. The GitHub page presently shows11,343separate authors and 995,819 commits (February 26 2021).

Measuring Collaboration

In addition to the number of contributors and commits, GitHub provides other insights.

git blame

And git blame will show us who authored each line of a file.

Showcasing Our Work

GitHub/GitLab allows you to show your abilities to prospective employers. This is especially useful for students, who can link to their GitHub from their resume.

If you are going to list Python as a skill on your resume, it is advisable to link to code you have written.

Link to Commits

If you make a commit to a git repository hosted on GitHub/GitLab, you can link to that specific commit.

For example, say you have a commit accepted in PyTorch, you can link to the commit on your resume and show everyone!

Technical Information

In the next couple slides we'll discuss how to interact with GitHub.

Adding a Remote

Let's say you have a local git repository, which you would like to push to GitHub. You first need to create a repository on GitHub, and then set that repository as a remote with:


	      $ git remote add origin [url provided by github]

You can list the remotes for a repository with:


	      $ git remote -v

Sending Code to GitHub

To send our commits in branch main to GitHub we use:


	      $ git push origin main

Getting Others' Changes

If there are changes in the main branch of the remote repo, you can run the following to incorporate them in your local repo:


	      $ git pull origin main

This runs a git fetch and git merge.

Pull Requests

If you have potential improvements to another person's repo, you can request to merge your improvements into their repo. You do this with a pull request. A pull request is not a part of git, but functionality provided by GitHub.
